# ショコラの記憶  
## サービス概要
チョコレートの思い出を、未来の自分へ残すためのアプリケーションです。  
食べたチョコレートの写真や味の評価を記録でき、大切な人へ贈った記録も残せます。  
次回購入時の参考に。チョコレートの記憶の振り返りに。
  
  
## 制作背景
私はバレンタインの催事の時期に、百貨店巡りを何年も続けていました。  
毎年同じブランドも出店する中で、異なる種類のチョコレートを試すことが楽しみでした。  
また、その楽しみを周囲のチョコ好きにもよくお裾分けしていました。  
  
しかし種類が多く、正確に「どんなチョコを食べたか」「味はどうだったか」「誰に何を贈ったことがあるか」などを覚えておくのは簡単ではありませんでした。  
これらを記録しておけるものがあれば、次回購入時の参考になり、その時の幸福感に浸れるのではないかと考え、本サービスが生まれました。    

主なターゲットはチョコレート好きの20〜40代の女性です。  


## 既存サービスとの比較・差別化ポイント
既存のアプリケーションには、実食記録に特化したものやSNSが主軸となるものが存在します。   
本サービスは、
- 実食用と贈答用を切り替えて記録を残しておける
- 他者との共有よりも、自己充足感に重きをおいている
- Webアプリケーション（インストール不要）のため、記録が増えても端末容量に依存せず長期的に記録を継続できる

という点で差別化を図ります。  

チョコレートの食べ比べを楽しんでいる方、大切な人へ贈ったチョコレートを記録として残しておきたい方々の一助になれば幸いです。
  

## MVP機能
#### 認証機能
- ログイン
#### 記録機能
- 登録（トグルで実食用/贈答用の切り替え）
  - 実食用：写真1枚、実食日、ブランド名、味評価、メモ
  - 贈答用：写真1枚、宛名、贈答日、ブランド名、メモ
- 一覧表示
- 詳細表示
- 編集
- 削除
  

## 本リリース追加予定機能
- 登録機能拡充
  - 写真複数枚（商品名なども写真で残せるように）
  - ★総合評価
  - カテゴリ選択（チョコ/ケーキ/焼菓子など）
- 検索/フィルタ表示（ブランド名、期間指定、カテゴリなどで絞り込み）
- トップページにランダムで労わる一言を表示
- UIの改善
  

### 技術予定スタック
|要素|技術|選定理由|
|-|-|-|
|バックエンド|Ruby on Rails|MVC構造が明確で迅速なWebアプリを開発できるため|
|データベース|PostgreSQL|Railsとの相性が良く、リレーショナルデータ管理に適しているため|
|UIデザイン|Tailwind CSS|ユーティリティファーストで効率的にデザインを構築できるため|
|認証|Devise|安全な認証機能を効率的に実装できるため|
|画像管理|Cloudinary|画像を外部保存でき、サーバー負荷を軽減できるため|
|開発環境|Docker|環境統一で将来的な保守性を高めるため|
