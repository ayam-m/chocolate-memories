# ショコラの記憶  
## サービス概要
チョコレートと向き合った時間を、未来の自分へ残す記録アプリケーションです。  
写真や味の評価を記録でき、大切な人へ贈った記録も残せます。  
  
  
## 制作背景
バレンタインの催事時期に、チョコレートのために百貨店巡りを何年も続けていた頃のことです。  
目的は自分用（食べ比べ用）と贈り物用でしたが、種類が多く、  
毎年正確に**どこの・どんなもの**を自分が食べたかor選んだかなどを把握しておくのは簡単ではありませんでした。  
徐々に記憶が曖昧になっていき、簡単に記録して遡れるものがあったらいいな、と感じていたのがきっかけです。  


## 主なターゲット層
- チョコ選びが好きで、周りの人に贈ったり、普段から自分へのご褒美にチョコレートを購入する20〜40代の女性  

自分自身がこの層に該当し、実際にバレンタインの百貨店催事でもたくさん購入しているのを見かける層なので、記録需要が高いと考えています。

## サービスの利用イメージ
記録系アプリケーションとして、自分用チョコ/贈答用チョコの記録をしていきます。 
- 曖昧な記憶を正確な情報として認識できます。  
- 味評価の記録をすることで、他の商品を試してみたくなる探究心を刺激します。  
- 記録が増えていくとアルバム的要素もあり、視覚的な自己充足感を体験できます。


## 既存サービスとの比較・差別化ポイント
本サービスは**未来の自分のための記録**に特化しています。  
チョコレートの購入・実食（評価）・贈答といった体験を、自分自身が振り返るために一貫して残せる設計としています。

既存サービスには実食記録を細かく記録できるものや、SNSが主軸のネイティブアプリケーションが存在します。  

#### 既存サービスと比較した強み
- 実食用と贈答用を切り替えて記録を残しておけるため、用途ごとの振り返りが容易
- 記録項目を最低限に絞ることで、気軽に継続できる設計
- 他者評価や共有を目的としないことで、自己充足感に重きをおいている
- Webアプリケーションのため、端末容量に依存せず長期的な記録に適している
  

## MVP機能
#### 認証機能
- ログイン
#### 記録機能
- 登録（トグルで実食用/贈答用の切り替え）
  - 実食用：
      - 写真1枚
      - 実食日
      - ブランド名
      - 味評価（5段階）：甘さ、濃厚さ、口どけ
      （味評価の項目は、見返した時に直感的に思い出せることを基準として厳選しています。）
      - メモ
  - 贈答用：
      - 写真1枚
      - 宛名
      - 贈答日
      - ブランド名
      - メモ
- 一覧表示
- 詳細表示
- 編集
- 削除

## 技術予定スタック
|要素|技術|選定理由|
|-|-|-|
|バックエンド|Ruby on Rails|MVC構造が明確で迅速なWebアプリを開発できるため|
|データベース|PostgreSQL|Railsとの相性が良く、リレーショナルデータ管理に適しているため|
|UIデザイン|Tailwind CSS|ユーティリティファーストで効率的にデザインを構築できるため|
|認証|Devise|安全な認証機能を効率的に実装できるため|
|画像管理|Cloudinary|画像を外部保存でき、サーバー負荷を軽減できるため|
|開発環境|Docker|環境統一で将来的な保守性を高めるため|
